TextProgress

TextProgress is a very simple text-based progress indicator. Insipored by terminal like proress bars it can be used to render simlpe progress.

Instalation

Installation using bower:

bower install --save text-progress
Or simply download text-progress.js and text-progress.css files and put them into your assets folder.

Usage

Create instance of TextProgress with one of predefined progress types.

• changes:


var changes = new TextProgress({
	type: {
		name: 'changes',
		total: 6,
		template: '${total} ${completed}${remaining}'
	},
	element: document.querySelector('#git-log')
});
changes.set(4);

• terminal:


var download = new TextProgress({
    type: {
        name: 'terminal',
        width: 10,
        charRemaining: '&minus',
        template: '[${percent}% ${completed}${remaining}]'
    },
    element: document.querySelector('#terminal-percent'),
    max: 100
});
download.set(0);

var inter = setInterval(function() {
    if (download.getPercent() >= 100) {
        clearTimeout(inter);
    }
    else {
        download.increment(1);
    }
}, 200);

What it's not

TextProgress is not progress bar in the strict sense. It doesn't have methods for animation or nice transitions. All it does is provide textual rendering of the current state of the process, meaning reprecentation of the current value, process etc. out of the total.

API

TextProgress uses UMD pattern to export itself either as CommonJS/AMD module or global constructor.

Instance object has few methods to control behavior:

  • set - Sets the current absolute value of the progress (not percent).
  • increment - Increment progress by specified value. Takes single numeric argument. Adds up to current value.
  • getPercent - Get current progress percent value.
